# 영속성 컨텍스트와 트랜잭션

> 오늘 받았던 수업과 질문에대해 잘 이해가 안가 따로 정리하기로했다. 

## 먼저 한줄로 요약 하자면 
- 영속성 컨텍스트는 엔티티를 관리하는 **메모리 공간** 이다. 
- **트랜잭션**은 작업을 하나의 단위로 묶어서 **성공하면 반영하고, 실패하면 되돌리는 범위**이다. 
- JPA에서는 보통 트랜잭션 안에서 영속성 컨텍스트가 엔티티를 관리하고, 트랜잭션이 끝날 때 변경 사항을 DB에 반영한다.

즉
> 영속성 컨텍스트는 “엔티티를 관리하는 곳”
> 트랜잭션은 “그 관리된 변경을 언제 확정할지 결정하는 범위”

이다.

## 영속성 컨텍스트란? 

영속성 컨텍스트는(Persistence Context)는 **엔티티를 영구 저장소(DB)에 저장하기 전에, 애플리케이션 쪽에서 엔티티를 관리하는 논리적인 공간**이다.

JPA에서는 EntityManager가 이 영속성 컨텍스트를 관리한다. 

쉽게 정리하면 
- DB에서 조회한 엔티티를 그냥 던져주는 게 아니라 
- 일단 메모리에서 관리하면서 
- 같은 엔티티를 재사용하고 
- 변경된 내용을 추적하고 
- 적절한 시점에 SQL로 반영하는 구조 

## 영속성 컨텍스트가 왜 필요한가? 
만약 영속성 컨텍스트가 없다면 
- 조회할 때마다 매번 새로운 객체를 만들어야 하고 
- 변경 여부를 개발자가 직접 추적해야 하고 
- insert / update / delete 시점을 직접 관리해야 하고 
- 객체 그래프를 다루기 매우 불편하다. 

JPA는 이걸 해결하려고 
**객체를 DB row처럼 즉시 다루는 방식이 아니라, 먼저 객체를 관리하는 계층을 둔 것** 이다. 

즉, 영속성 컨텍스트는 JPA가 **객체지향적으로 데이터에 접근하게 해주는 핵심 장치** 이다.
