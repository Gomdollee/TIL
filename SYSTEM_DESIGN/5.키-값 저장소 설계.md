# 키-값 저장소 설계
### 키-값 저장소 
**비관계형(non-relational) 데이트베이스**의 한 종류이다. 
- 데이터는 **key -> value 형태**로 저장 된다. 
- 이 관계를 키-값 쌍(pair) 이라고 한다. 

### 특징 
- key는 **유일해야 한다**
- value **key를 통해서만 접근 가능**
- value 타입은 제한 없음 

즉, 단순하지만 유연한 구조 

### 키 (Key)
- 일반 텍스트 가능 -> `last_logged_in_at`
- 해시 값 가능 -> `253DDEC4`

-> 성능상 **키는 짧을수록 좋다**

### 값 (Value)
값은 다양한 형태 가능 
- 문자열 (String)
- 리스트 (List)
- 객체 (Object)

-> 시스템 value의 구조를 신경 쓰지 않는다

### 대표적인 시스템 
- Amazon DynamoDB
- Memcached
- Redis

### 기본 연산 
키-값 저장소는 일반적으로 두 가지 연산을 지원한다.
 ``` java
put(key, value)
``` 
-> 데이터를 저장 
 ``` java
get(key)
``` 
-> key로 value 조회
